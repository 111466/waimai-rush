# 外卖冲冲冲 功能说明

本文档记录项目当前已经实现的游戏功能，便于后续开发、验收和需求拆分。

## 游戏概述

《外卖冲冲冲》是一款竖屏跑酷配送游戏。玩家自动沿城市道路前进，通过变道、跳跃、下滑和路口转向躲避障碍，同时完成取件和送件，尽可能获得更高收入、跑出更远距离。

游戏的核心循环是：

```text
自动前进 -> 躲避障碍 -> 取件 -> 根据路口提示选择方向 -> 送达 -> 获得收入和时间奖励 -> 难度提升
```

## 已实现功能

### 1. 城市路网系统

- 已实现基于 RoadGraph 的城市网格路网。
- 当前地图为 `5 x 5` 网格。
- 相邻路口间距为 `80m`。
- 每两个相邻路口之间存在双向道路。
- 玩家从底部中间路口出发，默认向前行进。
- 道路由节点和边组成：
  - 节点表示交叉路口。
  - 边表示道路段。
- 已实现根据当前道路方向计算玩家世界坐标、前进方向和朝向。

相关文件：

- `scripts/road_network.lua`
- `scripts/path.lua`

### 2. 三车道跑酷移动

- 每条道路包含左、中、右三条车道。
- 玩家默认位于中间车道。
- 支持左右变道。
- 变道过程带平滑动画。
- 玩家自动前进，不需要手动按前进键。
- 玩家速度会随着距离增加逐渐提升。
- 当前速度范围：
  - 初始速度：`8m/s`
  - 最大速度：`14m/s`

相关文件：

- `scripts/player.lua`
- `scripts/config.lua`

### 3. 跳跃和下滑

- 已实现跳跃动作。
- 已实现下滑动作。
- 跳跃和下滑有独立持续时间。
- 跳跃过程中可缓存下滑输入。
- 下滑过程中可缓存跳跃输入。
- 下滑时玩家模型会压低，角色身体、头部、帽子和配送箱位置会同步变化。

当前参数：

- 跳跃持续时间：`0.6s`
- 跳跃高度：`1.5m`
- 下滑持续时间：`0.5s`

相关文件：

- `scripts/player.lua`
- `scripts/input.lua`

### 4. 键盘和触屏输入

已实现键盘输入：

```text
A / 左方向键       左变道或左转
D / 右方向键       右变道或右转
W / 上方向键 / 空格 跳跃或直行选择
S / 下方向键       下滑
```

已实现触屏滑动输入：

```text
左滑 / 右滑  变道或选择转向
上滑         跳跃或选择直行
下滑         下滑
```

触屏滑动需要超过指定阈值才会触发，避免误触。

相关文件：

- `scripts/input.lua`

### 5. 路口转向系统

- 已实现路口区域。
- 每个路口有一个可穿越区域。
- 玩家进入路口区域后，可以选择：
  - 左转
  - 直行
  - 右转
- 普通道路上，左右输入用于变道。
- 进入路口后，第一次左右输入用于选择转向。
- 选择转向后，后续左右输入仍可用于调整出口车道。
- 直行时保持当前车道。
- 转弯时会根据玩家在路口内做出选择的时机决定出口车道。
- 已实现平滑转弯路径，路口内转向不是瞬间切换。
- 已修复进入路口时默认直行无路导致的死路误判：
  - 如果直行无路但左转或右转可走，不会立即游戏结束。
  - 只有当前路口没有任何可用出口时，才判定为死路。

相关文件：

- `scripts/path.lua`
- `scripts/intersection.lua`
- `scripts/road_network.lua`

### 6. 路口方向提示

- 玩家做出选择后，HUD 会显示已选择的方向。

### 7. 障碍物系统

已实现三类障碍物：

```text
block 普通障碍
- 不能跳过
- 不能下滑通过
- 需要变道躲避

low 低障碍
- 可以跳过
- 不能下滑通过

high 高障碍
- 可以下滑通过
- 不能跳过
```

其他已实现逻辑：

- 障碍物使用对象池创建和复用。
- 障碍物绑定到真实道路 edge 上。
- 障碍物会生成在玩家前方。
- 障碍物不会生成在路口安全区内。
- 玩家通过障碍物后，障碍物会被回收。
- 玩家转弯后会清理旧道路上的障碍物。
- 碰撞时会根据当前车道、跳跃状态和下滑状态判定是否失败。

相关文件：

- `scripts/obstacles.lua`

### 8. 难度成长

- 已实现基于行进距离的难度成长。
- 距离越远，玩家速度越快。
- 距离越远，障碍物间距越小。
- 后期有概率在同一排生成两个障碍物。

当前参数：

- 难度开始距离：`50m`
- 难度爬升距离：`400m`
- 障碍最小间距：`10m`
- 障碍最大间距：`20m`

相关文件：

- `scripts/config.lua`
- `scripts/obstacles.lua`
- `scripts/player.lua`

### 9. 取件系统

- 已实现取件点。
- 玩家未持有包裹时才会生成取件点。
- 取件点会生成在当前道路前方。
- 取件点随机出现在三条车道之一。
- 玩家需要在对应车道经过取件点才能成功取件。
- 成功取件后：
  - 玩家进入持有包裹状态。
  - 玩家身上的配送箱显示。
  - 系统开始准备生成送达点。
- 如果错过取件点，取件点会消失。

相关文件：

- `scripts/pickup_delivery.lua`
- `scripts/player.lua`

### 10. 送件系统

- 已实现送达点。
- 玩家持有包裹时才会生成送达点。
- 送达点会生成在远处可达道路 edge 上。
- 送达点随机出现在三条车道之一。
- 玩家需要在对应车道经过送达点才能完成配送。
- 成功送达后：
  - 增加收入。
  - 准时送达会增加连击数。
  - 迟到送达会按迟到时间扣收益并清零连击。
  - 清除持有包裹状态。
  - 隐藏玩家身上的配送箱。
- 如果错过送达点：
  - 送达点会消失。
  - 连击数清零。

当前奖励：

- 基础收入：`10`
- 连击奖励：`floor(comboCount * 0.5)`
- 迟到扣款：`floor(lateSeconds * 2)`
- 迟到收益允许为负数，会直接扣除总收入。

相关文件：

- `scripts/pickup_delivery.lua`

### 11. 订单倒计时和游戏结束

- 当前版本已移除全局倒计时。
- 取件并生成送达目标后，会启动当前订单倒计时。
- 订单时限会根据推荐路线距离和当前速度计算。
- 订单倒计时可以变为负数，负数表示已经迟到。
- 订单超时不会结束游戏，玩家仍然可以继续送达。
- 迟到越久，送达时扣除的收益越多。

已实现游戏结束条件：

- 撞到障碍物。
- 路口没有任何可用出口。
- 玩家选择了不存在的路口方向。

游戏结束后会显示结算面板。

相关文件：

## 当前游戏流程

```text
1. 初始化路网和场景
2. 创建道路、建筑、玩家、障碍物、取件点、送达点和 UI
3. 玩家自动向前移动
4. 玩家通过变道、跳跃、下滑躲避障碍
5. 玩家拾取包裹
6. 系统生成送达点并提供路口方向提示
7. 玩家在路口选择方向并继续前进
8. 玩家到达送达点，根据准时/迟到获得或扣除收入
9. 随距离增加，速度和障碍难度提升
10. 撞障碍或进入无可用出口路口时游戏结束
11. 显示结算面板，可重新开始
```

### 16. 多路线配送导航

- 已实现持久送件目标：取件后送达点会生成在远处可达 road edge 上，而不是只生成在当前道路前方。
- 送件目标建议距离为当前路线后续 `2-4` 个路口，极端情况下会回退到任意可达目标，避免无目标死局。
- 已实现基于 RoadGraph 的配送路线搜索模块：`scripts/route_navigation.lua`。
- 玩家偏离推荐路线后不会立即失败、不会额外扣时间、不会清空连击，系统会从当前位置到当前送件目标自动重规划。
- 只有玩家真正进入目标 edge 并越过目标点仍未送达时，才会按错过送达处理并中断连击。
- HUD 在路口继续显示实际可选方向；有配送路线时会额外显示建议方向，但不强制玩家选择。
- 已新增右上角路线小地图：
  - 显示简化 `5 x 5` 路网。
  - 高亮当前推荐路线。
  - 显示玩家所在路段/路口和送餐目标。
  - 错路后会短暂显示“已重新规划路线”。
