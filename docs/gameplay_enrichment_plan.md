# 《外卖冲冲冲》玩法丰富方案

## 目标

将当前“单取件点 + 单配送目标”的基础循环，升级为“地图多订单刷新 + 玩家自主路线选择 + 骑手成长 + 局内道具 + 任务成就”的配送跑酷体验。

核心体验保持不弹窗、不打断：

```text
随机城市路网生成
-> 地图刷新多个取餐点
-> 玩家通过小地图查看订单类型和奖励
-> 玩家驶向某个取餐点即接单
-> 导航生成送餐路线
-> 送达后获得收入、经验、连击和任务进度
-> 骑手等级提升后解锁更多订单数量、订单类型和成长内容
```

## 玩法循环

### 局内主循环

1. 每局开始后，系统根据骑手等级在路网中刷新多个取餐点。
2. 每个取餐点绑定一个订单，订单有类型、奖励、时限、预计距离和风险标签。
3. 玩家不通过弹窗选择订单，而是直接驶向目标取餐点。
4. 玩家拾取某个取餐点后，该订单成为当前订单，系统生成送达目标和推荐路线。
5. 玩家按导航送达，获得收入、经验、连击奖励和任务进度。
6. 未被选择的取餐点继续存在，超时后自动刷新。
7. 完成订单、升级、获得道具后，局内节奏继续推进。

### 局外主循环

1. 结算本局收入、经验、任务和成就。
2. 骑手等级提升，解锁更多订单数量和订单类型。
3. 使用长期收入升级能力。
4. 下一局获得更高收益目标和更丰富地图订单。

## 多订单系统

### 订单刷新规则

地图上同时存在多个取餐点。数量由骑手等级决定：

| 骑手等级 | 同时刷新取餐点 |
| --- | --- |
| Lv.1 | 2 个 |
| Lv.3 | 3 个 |
| Lv.6 | 4 个 |
| Lv.10 | 5 个 |

刷新原则：

- 取餐点优先生成在玩家前方或可达路线附近。
- 取餐点之间保持最小间距，避免多个订单重叠。
- 玩家持有订单时，不再接取新订单；未接订单仍可显示在地图上。
- 未接订单有接单倒计时，超时后刷新为新订单。
- 已接订单失败或完成后，系统补足地图上的取餐点数量。

### 订单类型

第一版订单类型如下：

| 类型 | 特点 | 奖励 | 时限 | 风险 |
| --- | --- | --- | --- | --- |
| 普通单 | 标准体验 | 标准 | 标准 | 标准 |
| 急送单 | 时间更紧 | 高 | 短 | 迟到惩罚高 |
| 远距离单 | 目标更远 | 高 | 较长 | 路线复杂 |
| 顺路单 | 距离短 | 低 | 宽松 | 适合保连击 |
| 易碎单 | 撞击失败 | 较高 | 标准 | 容错低 |

订单数据建议包含：

```lua
{
    id = 1,
    type = "rush",
    label = "急送",
    pickupEdgeId = 12,
    pickupEdgeDist = 38.0,
    pickupLane = 2,
    deliveryEdgeId = 44,
    deliveryEdgeDist = 26.0,
    deliveryLane = 1,
    reward = 18,
    xp = 12,
    timeLimit = 18.0,
    acceptTimeRemaining = 20.0,
    fragile = false,
    status = "available",
}
```

订单状态：

```text
available  可接取
accepted   当前进行中
delivered  已完成
failed     已失败
expired    未接取超时
```

## 小地图与 HUD

### 小地图

小地图是订单选择的主要入口，不使用弹窗。

小地图显示：

- 玩家位置
- 当前推荐路线
- 当前送达目标
- 多个可接取取餐点
- 不同订单类型的颜色或图标
- 当前订单状态信息

订单颜色建议：

| 类型 | 颜色 |
| --- | --- |
| 普通单 | 粉色 |
| 急送单 | 红色 |
| 远距离单 | 黄色 |
| 顺路单 | 绿色 |
| 易碎单 | 蓝色 |

小地图状态文字示例：

```text
可接: 普通 ¥10 / 急送 ¥18
当前: 急送 120m
目标 85m
已重新规划路线
```

### HUD

HUD 新增信息：

- 骑手等级与经验
- 当前订单类型
- 当前订单倒计时
- 当前道具
- 局内任务进度

建议布局：

```text
顶部：订单倒计时 / 收入 / 连击 / 速度
右上：小地图
右侧：当前道具
左侧：骑手等级与经验
底部或提示区：路口提示、导航建议、任务进度
```

## 骑手等级与成长

### 骑手等级

骑手经验来源：

- 完成订单
- 准时送达
- 连击送达
- 完成局内任务
- 解锁成就

骑手等级影响：

- 地图同时刷新订单数量
- 可出现的订单类型
- 高价值订单刷新概率
- 局外升级项上限

等级解锁建议：

| 等级 | 解锁内容 |
| --- | --- |
| Lv.1 | 普通单、顺路单，2 个取餐点 |
| Lv.2 | 急送单 |
| Lv.3 | 3 个取餐点 |
| Lv.4 | 护盾道具 |
| Lv.5 | 远距离单 |
| Lv.6 | 4 个取餐点 |
| Lv.8 | 易碎单 |
| Lv.10 | 5 个取餐点 |

### 局内成长

局内成长围绕连击：

- 准时送达增加连击。
- 迟到送达可完成订单，但清空连击。
- 撞击、错过送达点、易碎单破损会清空连击。
- 连击提高收入倍率。
- 连击达到指定值时，生成道具或提高高价订单概率。

连击奖励建议：

| 连击数 | 奖励 |
| --- | --- |
| x2 | 收入 +10% |
| x3 | 生成一个道具 |
| x5 | 下一批订单高价概率提升 |

### 局外升级

第一版局外升级项：

| 升级项 | 效果 |
| --- | --- |
| 起步速度 | 提高基础速度 |
| 最高速度 | 提高速度上限 |
| 操控能力 | 缩短变道时间 |
| 时间管理 | 增加订单时限 |
| 幸运派单 | 提高急送、远距离等高价订单概率 |

局外升级先以内存数据实现，后续再接本地存档。

## 局内道具

第一版只允许玩家持有 1 个主动道具。

### 道具列表

| 道具 | 效果 |
| --- | --- |
| 护盾 | 抵消一次正面碰撞 |
| 闹钟 | 当前订单增加数秒时间 |
| 导航针 | 短时间强化小地图路线提示 |
| 加速餐箱 | 短时间提高速度和收入倍率 |

### 道具来源

- 地图随机刷新
- 连击奖励
- 特定任务奖励
- 局外升级后开局携带

### 使用方式

第一版建议：

- 键盘：`E` 使用当前道具。
- 触摸：点击 HUD 道具按钮。
- 若玩家已有道具，再拾取新道具时直接替换，避免背包复杂度。

## 任务与成就

### 局内任务

每局刷新 3 个任务，作为短期目标。

候选任务：

- 本局完成 3 单
- 准时送达 2 单
- 完成 1 个急送单
- 连续送达 3 单
- 不撞击行驶 500m

任务奖励：

- 经验
- 收入
- 局内道具

### 长期成就

长期成就只触发一次。

候选成就：

- 完成首单
- 累计完成 50 单
- 累计准时送达 30 单
- 完成 20 个急送单
- 单局收入达到指定数值

成就奖励：

- 长期声望
- 经验
- 后续可扩展外观和称号

## 技术落地方案

### 订单模块

当前 `pickup_delivery.lua` 以单个取件点和单个送件点为核心。第一版改造为：

- 保留“当前订单”和“当前送达目标”的单目标导航逻辑。
- 新增可接订单列表 `availableOrders`。
- 取餐点视觉节点改为池化，支持多个取餐点同时显示。
- 玩家碰到某个可接取餐点后，将该订单设为 `activeOrder`。
- `activeOrder` 使用现有导航模块生成送达路线。

建议新增或调整接口：

```lua
pickup.TrySpawnOrders()
pickup.CheckOrderPickup()
pickup.CheckDelivery()
pickup.GetOrderTimerData()
pickup.GetMinimapData()
pickup.OnPlayerCollision(collisionType)
```

### 导航模块

`route_navigation.lua` 继续保持单目标路线，不需要支持多个送达目标。

规则：

- 未接订单不生成送达导航。
- 玩家接单后，调用 `nav.SetTarget()`。
- 玩家偏离路线后，沿用当前自动重规划逻辑。

### 成长模块

建议新增 `scripts/progression.lua`。

职责：

- 记录骑手等级、经验、总收入、升级项。
- 根据等级返回最大可接订单数量。
- 根据等级返回可刷新的订单类型。
- 提供局外升级加成。

核心接口：

```lua
progression.ResetRun()
progression.AddXP(amount)
progression.GetRiderLevel()
progression.GetMaxAvailableOrders()
progression.GetUnlockedOrderTypes()
progression.GetUpgradeBonus(key)
```

### 道具模块

建议新增 `scripts/powerups.lua`。

职责：

- 道具生成
- 道具拾取
- 当前持有道具
- 道具使用
- 限时效果更新

核心接口：

```lua
powerups.Init(scene)
powerups.Reset()
powerups.Spawn()
powerups.CheckPickup()
powerups.UseCurrent()
powerups.Update(dt)
powerups.HasShield()
powerups.ConsumeShield()
powerups.GetHUDData()
powerups.GetMinimapData()
```

### 任务成就模块

建议新增 `scripts/tasks.lua`。

职责：

- 管理局内任务
- 管理长期成就
- 监听订单、碰撞、距离等事件
- 生成 HUD 展示数据

核心接口：

```lua
tasks.ResetRun()
tasks.OnOrderAccepted(order)
tasks.OnOrderDelivered(order, onTime, reward)
tasks.OnOrderFailed(order, reason)
tasks.OnCollision(collisionType)
tasks.UpdateDistance(distance)
tasks.GetHUDData()
```

### 主循环接入

`main.lua` 中需要调整：

- 每帧调用多订单刷新。
- 碰撞后通知订单、任务和道具系统。
- 允许护盾抵消正面碰撞。
- 每帧更新道具状态。
- HUD 更新时传入成长、道具、任务数据。

主循环顺序建议：

```text
输入
速度/移动
路径推进
玩家位置
订单刷新与取餐检测
道具刷新与拾取
障碍生成与碰撞
送达检测
导航更新
订单计时
道具状态更新
任务成就更新
摄像机与 UI 更新
```

## 第一版验收标准

### 多订单

- Lv.1 开局地图上同时出现 2 个取餐点。
- 玩家驶向其中一个取餐点后，只接取该订单。
- 接单后小地图显示送达路线。
- 未接订单超时后能刷新。

### 订单类型

- 普通单、急送单、远距离单、顺路单、易碎单都能生成。
- 不同订单有不同奖励、时限和颜色。
- 急送单时限更短、奖励更高。
- 易碎单撞击后失败。

### 成长

- 完成订单增加经验。
- 骑手等级提升后，可刷新订单数量增加。
- 连击能提高收入。

### 道具

- 护盾能抵消一次正面碰撞。
- 闹钟能增加当前订单时间。
- 导航针能强化路线提示。
- 加速餐箱有持续时间，结束后恢复。

### 任务成就

- 局内任务能显示并更新进度。
- 长期成就只触发一次。
- 重开局后局内任务重置，长期成就保留在内存中。

## 开发顺序

建议按以下顺序实现，降低风险：

1. 多取餐点和多订单数据结构。
2. 小地图显示多个可接订单。
3. 接单后复用现有送达导航。
4. 骑手等级和经验。
5. 局内连击成长。
6. 护盾和闹钟道具。
7. 导航针和加速餐箱。
8. 局内任务。
9. 长期成就。
10. 局外升级和存档预留。

## 第一版边界

- 不做弹窗订单选择。
- 不做同时持有多个订单。
- 不做复杂背包。
- 不做真实存档，只保留内存长期数据。
- 不做订单详情大面板，小地图和 HUD 承担主要信息展示。
