# 《外卖冲冲冲》UI 原型设计说明

## 目标

本原型用于验证《外卖冲冲冲》的完整 UI 页面框架、局内 HUD 信息层级、局外成长入口和清爽商业跑酷视觉方向。

原型位置：

```text
prototypes/ui-flow/index.html
```

设计目标：

- 使用竖屏手机画布验证移动端布局。
- 页面可点击跳转，能体验主菜单、局内、暂停、结算、成长、升级、任务、成就和设置流程。
- 局内 HUD 不打断跑酷操作，不使用订单选择弹窗。
- 小地图继续承担多订单信息展示，订单选择仍通过玩家驶向取餐点完成。

## 页面结构

原型包含以下页面：

| 页面 | 用途 |
| --- | --- |
| 主菜单 | 开始配送，进入骑手、升级、任务、成就和设置 |
| 局内 HUD：可接订单 | 展示基础 HUD、小地图、多订单图例、任务和道具入口 |
| 局内 HUD：配送中 | 突出当前订单倒计时、导航路线和送达目标 |
| 暂停页 | 覆盖局内画面，提供继续、重开、设置和返回主菜单 |
| 结算页 | 展示收入、距离、订单、准时、连击和 XP 奖励 |
| 骑手成长页 | 展示等级、经验、解锁订单数量和下一等级奖励 |
| 局外升级页 | 展示起步速度、最高速度、操控、时间管理和幸运派单 |
| 任务页 | 展示本局任务进度与奖励 |
| 成就页 | 展示长期成就进度 |
| 设置页 | 展示音效、音乐、震动、操作方式和调试入口 |

页面流：

```text
主菜单 -> 开始配送 -> 局内可接订单
局内可接订单 -> 模拟接单 -> 局内配送中
局内配送中 -> 暂停 -> 继续
局内配送中 -> 模拟结算 -> 结算
结算 -> 再来一局 / 升级 / 主菜单
主菜单 -> 骑手成长 / 升级 / 任务 / 成就 / 设置
```

## 视觉规范

整体方向：清爽商业跑酷风。

核心颜色：

| 用途 | 颜色 |
| --- | --- |
| 主色 | `#FF8A1F` |
| 路线/成功 | `#2EE66B` |
| 急送/危险 | `#FF4D4D` |
| 信息/导航 | `#2F80ED` |
| 远距离/奖励 | `#FFD34D` |
| 深色 HUD 底 | `rgba(12,18,24,0.78)` |
| 卡片底 | `#F6F8FA` |

局内 UI 原则：

- 画面中心留给道路和障碍物。
- 顶部只放订单时间、收入、连击和速度。
- 左上放骑手等级与经验。
- 右上放小地图，小地图下方放订单图例。
- 底部放路口/导航提示。
- 右下放当前主动道具。
- 不在局内弹出大订单选择面板。

## 交互与动效

原型已包含以下交互：

- 主菜单进入局内。
- 局内模拟接单，切换到配送中状态。
- 局内模拟准时送达，回到可接订单状态。
- 局内模拟任务完成和获得道具，展示 toast。
- 局内暂停，显示暂停覆盖层。
- 结算页收入数字递增。
- 侧边原型导航可直接切换页面。

关键动效：

- 主菜单背景轻微视差。
- 页面短距离滑入和淡入。
- 小地图可接订单点脉冲。
- 配送中导航线流动。
- toast 弹入弹出。
- 结算收入数字递增。

## 验收标准

- 打开 `prototypes/ui-flow/index.html` 可直接查看原型。
- 桌面浏览器中显示居中的手机画布和右侧原型导航。
- 移动端宽度下隐藏右侧导航，手机画布铺满屏幕。
- 主菜单、局内、暂停、结算、成长、升级、任务、成就、设置都能进入并返回。
- 局内可在可接订单和配送中状态间切换。
- 小地图角色三角形、订单圆点、导航线、目标点和订单图例清晰可读。
- 竖屏 `390 x 844` 下没有明显文字重叠、按钮溢出或 HUD 遮挡道路中心。

## 后续接入建议

确认原型方向后，再拆成游戏内 UI 实现：

1. 先重构 `scripts/ui.lua` 页面状态，支持主菜单、运行、暂停和结算。
2. 再接入骑手等级、局外升级、任务和成就数据。
3. 最后将道具、任务完成、升级等奖励提示接成局内 toast。
4. SVG/PNG 素材在游戏内使用前需要按 UrhoX 资源加载方式转换或导入。
